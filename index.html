Questions

1) A client hands you a very wordy two-page Microsoft Word document and insists that the content must be converted to HTML and go on one particular screen in a website that's formatted for the size of an iPad screen. You and your supervisor agree that there's no way the entire content can fit on the screen at once and still be legible. What are some alternatives you might suggest to the client?

Solution 1

Suggest editing the content to filter out filler words and make more digestable for the web.

Solution 2

Take advantage of setting divs to scroll to include all of the content.

Solution 3

Include 'see more' links to toggle visability of addition content

2) You are creating an HTML page from scratch and you've encountered a problem where the CSS you've written and tested on your laptop is not working properly on a mobile device like an iPhone or an iPad. Give us some insight about how you might go about research and solving this problem, what resources would you use? What order? What would your process look like for solving the bug?

1.) Make sure<meta name="viewport" content="width=device-width" /> is included in the html document

2.) Check to see if the CSS is being included properly

3.) Run the code through a CSS Lint filter to check for errors/missing semi colons

4.) Make sure the media queries are set to the appropriate size in order to respond on changing platforms

5.) Make sure the browsers on the different platforms are up to date.

6.) Search Stack-overflow to possible solutions

7.) Try running the CSS in separate based on the size of the platform

8.) When in doubt, use jQuery to launch new CSS scripts when the width of the browser changes.
